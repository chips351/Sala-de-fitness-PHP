Proiect Sala de Fitness

Descrierea aplicației

Aplicația are rolul de a gestiona activitățile unei săli de fitness online. Permite administrarea utilizatorilor (admini, antrenori și clienți), gestionarea claselor de antrenament și a înscrierilor clienților la acestea.

Rolurile utilizatorilor:

  • Admin: creează, editează și șterge utilizatori, creează/editează/șterge clase, gestionează abonamente și plăți, vizualizează statistici.
  • Antrenor (Trainer): creează/editează/șterge clase proprii, vizualizează clienți înscriși.
  • Client: se poate înregistra și loga, vede clasele disponibile, se poate înscrie/retrage de la clase, vizualizează abonamentele și programul personal.

Creearea bazei de date și a tabelelor

CREATE DATABASE IF NOT EXISTS fitness_db;
USE fitness_db;

CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    role ENUM('admin', 'trainer', 'client') NOT NULL,
    phone VARCHAR(20),
    status ENUM('active', 'inactive') DEFAULT 'active'
);

CREATE TABLE classes (
    id INT AUTO_INCREMENT PRIMARY KEY,
    trainer_id INT NOT NULL,
    title VARCHAR(100) NOT NULL,
    description TEXT,
    date DATE NOT NULL,
    time TIME NOT NULL,
    duration INT,
    max_clients INT DEFAULT 10,
    location VARCHAR(100),
    FOREIGN KEY (trainer_id) REFERENCES users(id)
        ON DELETE CASCADE ON UPDATE CASCADE
);

CREATE TABLE class_registrations (
    id INT AUTO_INCREMENT PRIMARY KEY,
    class_id INT NOT NULL,
    client_id INT NOT NULL,
    registered_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (class_id) REFERENCES classes(id)
        ON DELETE CASCADE ON UPDATE CASCADE,
    FOREIGN KEY (client_id) REFERENCES users(id)
        ON DELETE CASCADE ON UPDATE CASCADE
);

CREATE TABLE subscriptions (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    type ENUM('Basic', 'Premium', 'VIP') NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    status ENUM('active', 'expired') DEFAULT 'active',
    FOREIGN KEY (user_id) REFERENCES users(id)
        ON DELETE CASCADE ON UPDATE CASCADE
);
